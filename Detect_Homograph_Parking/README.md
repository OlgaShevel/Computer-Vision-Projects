
# Проект определения занятости парковочного места по снимкам с камер

### Условия задачи
:
Объект: Плоская уличная парковка. 
Цель: Определить статус (занято/свободно) конкретного парковочного места на основе изображения с камеры.
Масштабируемость: В данной реализации используется одна камера, но архитектура поддерживает любое количество камер (каждая требует отдельного файла калибровки).

Входные данные:
GeoJSON (park_<idx>.geojson), где park_idx - номер тестируемого парковочного объекта.: Полигоны парковочных мест в условных реальных координатах. 

Калибровка (calibrate_< camera _idx>.json), где camera_idx - номер камеры. Привязка реальных координат к пикселям на изображении: минимум 4 точки для гомографии/перспективы.

Тестовые изображения в формате test_<camera_idx><park_idx><test_idx>.jpg, где test_idx - номер теста.

Выходные данные:
JSON-файл (result_....json), содержит статус занятости (detected: true/false) для тестовых файлов.

{
  "params": {
    "park_idx": 30,
    "calibrate_idx": 1
  },
  "result": {
    "1": {
      "detected": true
    }
  }
}



### Решение

Суть проекта:<b>
Соединение нейросетевого алгоритма детекции с геометрической трансформацией (гомографией).

<b>Детекция:<b>
Модель на базе YOLOv8 (ONNX) обнаруживает автомобили на изображении и возвращает ббоксы в пиксельных координатах.

<b>Пространственная привязка (Homography):<b>
Матрица гомографии H, рассчитанная по 4 опорным точкам, позволяет преобразовать координаты детекции из плоскости изображения в плоскость парковочной схемы.

<b>Логика определения:<b>
Алгоритм проверяет пересечение центра детектированного автомобиля с полигоном конкретного парковочного места. Если центр бокса попадает в полигон — место считается занятым
